Welocome to our Project

This project primarily used the embedded systems microcontroller MSP432P401R.

The project comprised of using a master MSP and slave MSP via connected through bluetooth, one giving the instructions for direction and the slave sending the receiving data to the speed controller and outputting the chirp to the two motors. There was a source of power using 12 Volts.

Conclusion

The Electrokart features full functioning omnidirectional capabilities and extensive firmware/ software. Using Bluetooth modules connected via UART (Universal asynchronous receiver transmitter) allow the master and slave to communicate. The master is comprised as the wireless remote control with the joystick conducting the direction. The slave is used to register the values from the master and output PWM (Post width modulation) waves to the speed controller. Once the speed controller reads the frequency and duty cycle of the PWM waves, it transfers the chirp to the motors. In order to log data, another Bluetooth module was connected on the master and through UART is transferred to RealTerm in order to show if the connection was successful.

The circuitry of the Electrokart includes crucial elements for it to function. The battery was the hardest to gauge, it was outputting 12 volts with 18Amps. When connecting the battery to the circuit, a power train for power and ground connected the fuses which were attached to the speed controller (in case it blew or shorted). The speed controller was connected to polarity switches which means that when the cart moved forwards we could reverse the direction by flipping the switches.
